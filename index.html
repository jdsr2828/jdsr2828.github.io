第二篇博客

发表于 2024-09-30

​ 每个不曾起舞的日子，都是对生命的一种辜负

​ Hello各位，这是我的第二篇博文，本篇文章我将讲述一下关于算法竞赛的流程以及如何去提升自己的算法构思与实现能力（你甚至不需要参加真实的算法竞赛），首先，算法竞赛分为两种赛制oi和acm赛制，我们来看一下oiwiki上对算法竞赛的定义

OI（Olympiad in Informatics，信息学奥林匹克竞赛）在中国起源于 1984 年，是五大高中学科竞赛之一。

ICPC（International Collegiate Programming Contest，国际大学生程序设计竞赛）由 ICPC 基金会（ICPC Foundation）举办，是最具影响力的大学生计算机竞赛。由于以前 ACM 赞助这个竞赛，也有很多人习惯叫它 ACM 竞赛。

​ 笼统上而言，oi赛制指的是你只能提交一次最终通过特定样例评判你是否得到了一个正确的程序，而acm能提交多次，但会产生一定的罚时。在主流竞赛中，icpc、ccpc、团队设计天梯赛是acm赛制，而蓝桥杯，csp，pat认证赛制是oi赛制，通常而言oi赛制对选手提交程序的精确性有更高的要求，且通常不允许携带任何纸质资料，而acm比赛则在难度上更为艰难。

​ 现今比赛除了蓝桥杯还对语言具体分组以外，其他所有比赛都允许使用c、c++、java、python这几种主流编程语言，而通常c++是最通用的也是蓝桥杯中含金量最高的分组，因为c++语言的一些特性比较适合作为算法竞赛的常用语言，包括运行时间更快，更贴近底层以及更容易区分算法实现时间复杂度对最终比赛结果的影响。

​ 学习算法竞赛能带给我们以下好处：
⚪更好的了解一门语言的特性

⚪拥有更好的分析能力，能看清楚事物间的联系与共同性质，从混沌中寻找秩序，并抽象成具体的数据结构以及使用更能反映问题本身性质的算法

⚪更注重细节与逻辑，通过控制流程来实现对所有情况不重不漏的考虑

⚪能更好的阅读代码，避免重复造轮子，当然初期尽量多造轮子来锻炼自己分解步骤并寻找一些相同点的能力（很重要的一点，在学校里你听的都是前人造过的轮子，则以后的工作或研究中你需要自己摸索并开辟出新的道路）

⚪你将比同龄人有更加清晰的头脑，提前接触一些理论计算机（tcs）的知识，知道一些计算机行业新新事物的本质，从而做出更好的判断（张爱玲曾经说过“出名要趁早”，许多acmer都是零基础，但即使是现在开始学习也不迟）

参加算法竞赛能带给我们以下好处：

⚪拥有一群志趣相同的朋友

⚪如果能得奖，这将是非常具有权威性和说服力的，你会获得比别人更好的工作机会

​ 下面我将讲解一个基本的概念来展现一个acmer是如何思考问题的，就拿c语言指针来说，指针存储两个，一个是其指针内的数据，另一个是指针内的地址，（java不具有指针因为它默认所有实现类的赋值时给予其地址（相当于加了取地址符&），类中元素调用时以结构体方式调用其内部数据（相当于加了寻址运算符*），实际也是个结构体指针，所以c语言中不允许非指针结构体嵌套也是这个原理，因为构造结构体时需要申请其内部所有元素的空间，比如

Struct a

{

Int b;
a c;

};

​ 嵌套后编译器会首先定义a类结构体，然后立刻申请a类结构体c的空间，计算c所需要的空间需要计算c所具有的所有元素的空间，即c.c以及c.c.c（这里不存在二义性）乃至无穷，无限申请空间直至发生栈溢出也无法返回一个有效的空间值，而指针结构体嵌套不会，因为其只拥有存储地址的空间，所以指针相当于一个缓冲作用避免了栈溢出）。而寻址符（*）使计算机瞬移到该数据表示的位置。我们通常需要将指针指向一个值或另一个指针，而后者如果采用结构体指针可以构成一个链表，指针还具有可叠加性即具有二重或多重指针，而二维或多维数组通常是通过这种方式实现的，在使用指针方式申请数组时，可以通过重载定义的自增运算符++在同一维度下移动或直接+n表示取到第n-1行，使用寻址符来移动到下层指针，直到取到数位置。

第1.5篇博文

发表于 2024-09-29

Hello各位这是我的第1.5篇博文，我主要想和大家介绍一下如何将我们专业学的东西与本学校的学科特色相结合。

​ 众所周知，我们学校是一个财经类院校，软科排名远大于校友会排名，所以至少能看出我们学校实际上是个偏应用型的学校，也就是说从我们学校读出要么就业要么就当考研跳板。而从我们学校最近的动向来看，包括对智能科学与技术、数字经济卓越类专业的增开，学校对于人工智能方面的发展愈发重视，但首先从定位来看，并不是工业型的人工智能，而是财经类的人工智能，因此可以合理利用学校所带来的资源（将个人命运与集体命运紧紧联系在一起），多进行对于该方面的学习。

​ 既具备人工智能领域知识，又具备财经背景知识的学科领域有：博弈论（首选，可以详细看数字经济专业培养方案），网络经济学（社交网络、信息网络、供应链网络都和财经密不可分，且同时不是硬工科背景），一些数据驱动的学科研究如计量心理学、生物信息学（具有统计学背景，本校有许多老师研究）。

​ 那如何去培养这些学科的素养，一种好的方法就是我之前提到过的算法竞赛（这对于所有专业其实都大有裨益，后面我会讲到）。众所周知，我们学校存在官僚主义盛行的毛病，各种学生会自律委大行其道，人们更偏向于追求虚无缥缈的党争而对于实际的学术能力培养嗤之以鼻，当然如果未来你想成为这样的人，那我想告诉你你的晋升渠道其实没有这么宽。而对于算法竞赛乃至于各种其他学科竞赛，既是一种培养学术氛围和人际交往能力的方式，也是一种对于这种不良风气的无言的声抗。所以与其把大学生涯花费在这种事上，还不如花在有意义的逻辑思维的体操和专业能力的训练上（你想想，你参加这种社团活动或学生会活动本质上就是为你以后做管理层做准备，如果你未来相当交际花那确实有用，可大多数人还是得靠自身能力吃饭，更何况很多人都没有很多硬关系）。还有一些所谓的挑战杯或市场调研等创业大赛，我想说你搞不过那些财经专业的，毕竟我们学校就是财经类大学，不妨看看财经专业和信息专业的名额比重，更何况你没有那些水平更高的学校的包装能力与技术支撑。

​ 然后谈一下数学竞赛，本人是去年大学生非数学A类省三，基本没有特别训练，我根据我所了解的情况说一下，我觉得这是一个比较重要的比赛，它能锻炼你的一些化归能力即如何把一些奇形怪状的公式划归一些典型的标准型，也是对于高维思考能力的锻炼，可以着重看一下微积分中的曲面曲线积分部分（格林公式，stokes公式）以及收敛中的一致收敛（函数列收敛）部分，这部分高等数学不会细讲，然后泰勒展开，指对互化这种硬计算对于机器学习、数值计算方面的也尤为重要，其他的部分可能不是特别重要，但要有良好的几何直观概念。数学可以几乎解决所有问题，但它要么是一个全局概括性的解决（各种基础性的定义与反例的攻击），要么是一些对于具体问题过于细致的解决（各种具有非常多参数的微分方程）。诚然我们需要有，但遗憾的是，大多是数学知识解决不了实际问题，而解决数学竞赛问题需要学习许多与专业本身无关的知识，如果学习过高数的同学可能会发现我们并不在乎对数学理论的分析，而更注重于各种微积分的变换，而这些变换完全可以被一些更高级的理论降维打击，如果各位有机会学习到抽象代数会发现这仅仅是一些更一般情况的冰山一角，我们学的是计算机，而不是计算。我们不是纯数专业的学生，无需对一些数学模型给出显式的计算公式，更何况99%的人都没有良好的数学直觉和训练使得我们对于每一道题都能一眼看出解法。我们要使用计算机去解决实际问题，而参加算法竞赛既能让你掌握一门语言的精妙之处（通常是对一些重复问题的模块化集成），也能横向比较各种语言的优劣之处，更能快速的用一套现成的流程去更快的对现实问题进行建模，如果能结合你高数所学内容，你会发现你很容易对于一个数学问题进行建模，而我们还有数学建模比赛（这个比赛学校很重视），实际上算法比赛中的很多算法都是数学建模中那些高大上算法的基本组成部分（可以看一下平面最近点对和层次聚类、voronoi图和kmeans聚类），当你达到一定的境界，你甚至能创造属于你自身的算法。

​ 下篇我们会讲到有关算法竞赛的选择和比赛流程以及怎样学习算法的内容，敬请期待。

第一篇博文

发表于 2024-09-28

Hello大家好，这是我的第一篇博客，我想主要讲一我的学习经历以及对于算法竞赛的一些理解

​ 我在初二的时候参与了noip的普及组初赛，当时拿的是二等奖，由于本人是浙江省，常年分数线是各省份最高的，所以和当时的同龄人相比不是特别出色。高中时期我参加了数学竞赛和一些国际性的数学竞赛，我比较感兴趣的是各学科间的交融与其在数理方面的共识。

​ 对于我而言，算法竞赛是比较能体现本质的东西，它相对于我们大学生期间能参加的其他竞赛更具有专业性和普适性。

​ 大学生主要能参加的对于我们专业理解有意义竞赛大概是这几种：

​ 电赛、大数竞赛、大学生数学建模大赛、挑战杯、计算机设计大赛、华为ict等等

​ 其中电赛、计算机设计大赛是比较偏向于应用，强调对专业全局的理解以及对于当今计算机行业应用发展的了解，华为ict这种企业举办的比赛相对于是对一个实操的考验，大学生数学建模大赛则是对数学领域的综合应用（对志向深入学习计算机乃至读研的同学可能尤为有用，我后面会讲到），大数竞赛则是对于数学理论的了解，它的运作模式和算法竞赛比较像，后面我会讲到。

​ 然而当今时代我们需要的是更加偏向软件方面的人才，那算法竞赛就是对于软件制作的一种具象化。我们要解决通过计算机解决一个问题，我们首先要在数学层面给出一个合适的算法，衡量一个算法的好坏有时间和空间两个层面，有时我们会将这个行为和编程同时进行，构成一个反馈的回路。

​ 然而对于我们学校来说，我们很多课程过于强调理论，以至于很多虽然获得很高分数的人实际不具备良好的实操经验（或者说他们更注重于计算机的应用而不是对于其底层模式的理解和开发），参加这个比赛可以使我们在兼顾学业的同时学到更多关于计算机软件开发的底层模式（通常是一些更为抽象，更加契合软件开发复杂性的知识），同时未来也能拥有更多展现的舞台（即更多工作的机会）。

​ 下一期我会主要讲一下如何将所学专业和本学校的学科特色相结合，大概更新在国庆节以后了
